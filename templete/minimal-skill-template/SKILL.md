# Your Skill Title

## Overview

说明这个 Skill 解决什么问题。

建议补齐：

- 目标任务是什么。
- 适用范围是什么。
- 明确不适用的场景是什么。

## Process

按阶段写出模型应如何执行：

1. 先识别用户意图和任务边界。
2. 再确认输入、约束和目标输出。
3. 按既定流程完成任务。
4. 如遇异常或边界情况，切换到相应分支处理。

## Rules

写关键规则时，优先说明“为什么重要”。例如：

- 先确认输入是否完整，避免建立在错误前提上继续输出。
- 对边界场景要明确说明处理方式，不要自行扩展用户未要求的能力。
- 如果任务中存在高风险内容，应先提示限制，再继续安全范围内的处理。

## Output Format

如果输出有稳定格式，请在这里给出模板。例如：

```markdown
# 标题
## 摘要
## 关键内容
## 建议或下一步
```

如果输出没有固定模板，也应说明：

- 输出至少要包含哪些部分。
- 命名规范是什么。
- 成果交付时重点检查什么。
